  2. How To Use:
    • Login Information:

      Account(A), Passowrd(P)

      • Administrator: A: admin@qq.com P: 1
      • Manager: A1: p@qq.com P1: 1.     A2: manager1@qq.com P2: 1
      • Regular User: A1: userA@qq.com P1: 1     A2: userB@qq.com P2: 1    A3: userC@qq.com P3: 1
      • Only one index link has been given due that the home page is different for different type of users. "user A" and "user B" have applied the job "Senior Los manager" managered by manager "p@qq.com"
    • Unregistered User:

      A basic home page just for searching jobs will be shown by clicking http://dwarf.cit.griffith.edu.au/~s2813995/wp/assign2/index.php. In the simpliest searching page, there are login or register interfaces for user to authenticate or register to be a new member. The searching function supports keywords, category, location and job type searching. In the "latest jobs" part, jobs which were posted in the nearest 5 days are listed directly. If click the "view more recent jobs", jobs posted in recent 10 days will be seen. In the "Brows Jobs By Category" user can easily brows all the jobs belong to that category. By clicking the "register", the register page will be seen. The required fields are labled "*". A user can upload his or her photo by choosing a file, but it is not mandatory. Every job title is a link leading to the corresponding job details. There is a "Apply this job" button in job detail page. Only login user can apply a job.

    • Regular Logged in User:

      Using a regular use account, two main functions are added inaddition to a unlogin user, which are "Update the Password" and apply a job. A the head of each page, a remainder is remainning who is using it at that time. And the user can logour at any time.

    • Manager:

      In assignment1, every user can see the employer page. In this solution, only the manager can see the employer page. If login as a manager, the home page will be changed into the employer's page. The manager can see the posted job by that employer. The manager has the right to post a new job. And there is a "Global seach" button in the employer page, which can give the manager a job searcher's view. Each job title in the employer page is a link leading to the details of that job. The manager can delete or update the job. In the job detail page, the manager can see the applicants who have applied this job.

    • Administrator:

      Similar to the manager, only login using an administrator's account can the user see the adminstrator page. In the adminstrator page, a new manager can be asigned using his or hers email and the employer's name.

  3. Critical Techniques:
    • Ajax

      The login process and the update password process are implemented by using a pop-up dialog. The tiny form in the dialog is loaded by using jQuery. ajax() fucntion sends the user authentication details to the server.

    • Validation

      The user's input has been validated on both server and client sides. At the client side, javascript has been used to implement validation. At the server side, validation are realized by using ajax with returning value from php "echo".

    • Transaction Processing

      Transaction has been implemented for multiple users concurrently accessing and updating the same database.

    • Foreignt key constraint

      Foreign key constraint has been included in the database design(can be seen in the sql statements). And the delete constraint are implemented by using cascading delete method.

  4. Userbility Test:

    I used DialogDesign usability test method to implement the userbility test. It consists of three phases: Interview, Solving test tasks, and Debriefing. Initially, there is only a keyword search in the searching page. But the user prefer to seach the job by some specific rules. Then the system adopted the job type, location and categoriy search. The users prefer borders with round corner. For all the borders of the web site are changed to round corner.
